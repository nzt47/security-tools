
# 任务6：核心链路集成测试

## 阶段目标

编写端到端的集成测试，覆盖系统核心业务链路，验证跨模块交互的正确性和数据一致性。

## 背景信息

系统已有部分集成测试在 `tests/integration/` 目录下，但核心链路的端到端覆盖不足。需要针对以下关键链路编写集成测试：
1. 对话处理全链路
2. 熔断降级联动
3. 记忆读写链路
4. 容灾恢复端到端验证

## 技术要求

**集成测试基本原则：**
1. 使用真实模块交互，尽量少mock
2. 测试数据使用临时文件/内存数据库，不影响真实数据
3. 每个测试用例前后清理环境
4. 使用 `@pytest.mark.integration` 标记
5. 测试超时设置合理（默认30秒）

**对话处理全链路集成测试（8个用例）：**
- 正常对话流程：输入→模型→响应→记忆存储
- 带工具调用的对话流程
- 多轮对话的上下文一致性
- 对话过程中触发降级的行为
- 对话过程中触发熔断的行为
- 对话过程中限流的处理
- 异常输入的全链路处理
- 对话取消/中断的资源清理

**熔断降级联动集成测试（6个用例）：**
- 熔断触发→自动降级→恢复的完整流程
- 多级降级的顺序正确性
- 降级后功能的可用性（不是完全不可用）
- 熔断恢复后的功能自验证
- 降级状态的监控告警触发
- 手动干预熔断/降级状态

**记忆读写链路集成测试（6个用例）：**
- 并发读写记忆的数据一致性
- 记忆存储→检索→删除的完整生命周期
- 敏感信息在记忆链路中的过滤效果
- 记忆持久化的可靠性（进程重启后数据存在）
- 记忆容量限制的处理
- 记忆搜索结果的排序正确性

**容灾恢复端到端测试（6个用例）：**
- 配置文件损坏→自动修复→服务恢复
- 消息文件损坏→逐行恢复→数据完整性
- 数据库损坏→重建→数据迁移
- 服务崩溃重启→状态恢复→功能自检
- 部分模块故障→降级运行→逐步恢复
- 容灾过程中的数据零丢失验证

**API网关限流全链路测试（5个用例）：**
- 全局级限流的全链路验证
- 用户级限流的隔离性
- 接口级限流的精准性
- 限流触发后的响应格式正确性
- 限流解除后的自动恢复
