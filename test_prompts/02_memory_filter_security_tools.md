
# 任务2：记忆敏感过滤与安全工具全面测试

## 阶段目标

为 `agent/memory/filter.py` 和 `agent/security_utils.py` 编写全面的测试用例，覆盖敏感信息过滤和安全工具的所有边界场景，确保数据安全性。

## 背景信息

云枢系统的记忆模块需要对用户数据进行敏感信息过滤，防止密码、令牌、身份证号等敏感信息被持久化存储。安全工具模块提供了各类安全相关的工具函数。
- 敏感信息过滤是合规性要求，必须100%可靠
- 安全工具模块覆盖率目标：≥95%
- 记忆过滤模块覆盖率目标：≥90%

## 技术要求

**必须遵循的编码规范：**
1. 所有测试文件放在 `tests/unit/` 目录下
2. 使用 `pytest` 框架，使用 `@pytest.mark.unit` 和 `@pytest.mark.p0` 标记
3. 测试用例必须有清晰的中文注释
4. 敏感数据测试使用模拟数据，禁止使用真实敏感信息

**敏感信息过滤测试清单（20个用例）：**
- 密码过滤：各种格式的密码（明文/Base64/哈希）
- 令牌过滤：API Key、JWT Token、Access Token
- 身份证号：18位和15位身份证号
- 银行卡号：16-19位银行卡号
- 手机号：中国大陆手机号格式
- 邮箱地址：各种邮箱格式
- 多敏感信息混合：一条消息中包含多种敏感信息
- 部分脱敏验证：只脱敏敏感部分，保留上下文
- 空输入/None输入的处理
- 超长文本（>10KB）中的敏感信息检测
- 编码绕过：URL编码、Unicode编码的敏感信息
- 大小写变体：PASSword、PassWord等
- 分隔符变体：p@ss、p-ss等
- 误报率验证：正常文本不应被误过滤
- 嵌套结构中的敏感信息（JSON/列表/字典）
- 自定义敏感词的添加和删除
- 过滤日志的完整性（记录过滤了什么）
- 过滤性能：10KB文本处理时间<100ms
- 过滤后数据的可逆性（不可还原）
- 多语言环境下的过滤效果

**安全工具全面测试清单（25个用例）：**
- 输入验证：XSS注入检测
- 输入验证：SQL注入检测
- 输入验证：命令注入检测
- 文件路径安全：路径穿越检测
- 文件类型校验：魔数验证
- 字符串安全：安全的字符串比较（防时序攻击）
- 随机数生成：密码学安全的随机数
- 哈希计算：MD5/SHA256/SHA512
- 加密解密：对称加密的正确性
- 签名验证：数字签名的正确性
- 证书验证：SSL证书有效性检查
- CSP策略生成：内容安全策略
- CSRF令牌生成与验证
- 速率限制辅助函数
- 安全Headers生成
- 输入长度限制校验
- 字符集白名单校验
- HTML标签过滤（防XSS）
- URL重定向验证（防钓鱼）
- 序列化数据安全校验
- 异常信息脱敏（错误消息不泄露敏感信息）
- 安全日志格式规范
- 权限检查辅助函数
- 会话ID生成安全性
- 密码强度评估函数
