
# 任务1：熔断器与限流器边界测试

## 阶段目标

为 `agent/circuit_breaker.py` 和 `agent/rate_limiter.py` 编写全面的边界值测试用例，确保核心稳定性模块覆盖率达到90%以上，覆盖所有异常路径和边界条件。

## 背景信息

云枢系统是一个智能代理平台，熔断器和限流器是系统稳定性的核心保障模块。
- 熔断器已有混沌测试：`tests/chaos/test_circuit_breaker_chaos.py`
- 限流器已有基础测试，但边界覆盖不足
- 测试框架使用 pytest，配置在 `pytest.ini`
- 覆盖率目标：两个模块均≥90%

## 技术要求

**必须遵循的编码规范：**
1. 所有测试文件放在 `tests/unit/` 目录下
2. 使用 `pytest` 框架，遵循项目现有测试风格
3. 测试用例必须有清晰的中文注释说明测试场景
4. 使用 `@pytest.mark.unit` 和 `@pytest.mark.p0` 标记
5. 优先使用 `unittest.mock` 进行依赖隔离

**熔断器边界测试清单（15个用例）：**
- 空请求/零请求时熔断器状态（不应熔断）
- 刚好达到最小请求数时的边界行为
- 错误率刚好等于阈值（30%）的边界
- 错误率略高于/低于阈值的边界
- 冷却时间刚好到达的边界
- 半开状态最大试探请求数边界
- 并发请求下的熔断器状态一致性
- 时间窗口滚动时的计数准确性
- 异常类型过滤（只对特定异常熔断）
- 熔断器名称/配置为空的情况
- 重置功能的正确性
- 多级熔断器（嵌套调用）的行为
- 熔断器指标统计准确性
- 快速失败时的错误码规范性
- 熔断器状态转换的日志完整性

**限流器边界测试清单（12个用例）：**
- 零阈值/零窗口的边界处理
- 刚好达到限流阈值的边界
- 令牌桶算法的令牌生成准确性
- 突发流量下的限流准确性
- 分级限流（用户级/接口级/全局级）的优先级
- 限流后的错误响应格式
- 限流计数器的并发安全性
- 时间窗口重置的准确性
- 不同IP/用户的限流隔离性
- 限流白名单/黑名单功能
- 动态调整限流阈值的正确性
- 限流指标上报的准确性

**异常路径验证要求：**
- 所有业务异常必须携带明确错误码
- 异常消息包含 trace_id 等上下文信息
- 异常路径必须输出结构化日志（JSON格式）
- 禁止静默返回 None 掩盖错误

## 预期成果

1. 新增文件：`tests/unit/test_circuit_breaker_boundary.py`（≥15个测试用例）
2. 新增文件：`tests/unit/test_rate_limiter_boundary.py`（≥12个测试用例）
3. 两个模块的单元测试覆盖率均≥90%
4. 所有测试可独立运行，不依赖外部服务
5. 测试执行时间总和≤30秒
