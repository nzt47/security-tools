## 1. 项目概述

### 1.1 项目背景

云枢 (Yunshu) 是一个拥有完整「感知-认知-行动」闭环的数字生命体项目。该项目旨在创建一个能够感知自身运行状态、理解用户意图并做出智能响应的自主对话系统。

### 1.2 项目目标

- **自主感知能力**：实时监测 CPU、内存、电池、磁盘、网络等系统状态
- **智能认知能力**：拟人化地理解和表达自身状态
- **智能行动能力**：根据状态变化调整行为，拒绝危险操作
- **完整记忆系统**：长期记忆、滚动摘要、黑匣子日志
- **人格化交互**：具备人格特征和个性化响应

### 1.3 主要功能

| 功能模块 | 描述 |
|---------|------|
| 感知层 | CPU/内存/电池/磁盘/网络/温度等状态采集 |
| 认知层 | 传感器数据拟人化翻译、提示词管理 |
| 记忆层 | 对话历史、滚动摘要、黑匣子日志、向量记忆 |
| 行为层 | 行为模式控制、任务拒绝、行为降级 |
| 权限系统 | 危险操作识别、自动备份、二次确认 |
| LLM 集成 | 支持 OpenAI、Anthropic 等大模型 |
| 规划引擎 | 复杂任务分解与执行 |
| V2 增强 | LifeTrace、Persona、蒸馏学习 |
| 语音交互 | TTS/STT 语音识别与合成 |
| OCR 能力 | 屏幕内容识别 |
| 快照管理 | 系统状态保存与恢复 |

### 1.4 应用场景

- 个人数字助手
- 智能客服
- 教育辅助工具
- 自动化任务执行
- 健康监测助手

### 2.2 技术栈

#### 2.2.1 核心技术

| 层级 | 技术 | 用途 |
|-----|-----|------|
| **语言** | Python 3.x | 主要开发语言 |
| **前端** | React + TypeScript + Vite | Yunshu UI 界面 |
| **向量库** | ChromaDB | 向量存储与检索 |
| **LLM** | OpenAI API / Anthropic API | 大语言模型集成 |
| **语音** | pyttsx3 / SpeechRecognition | TTS/STT |
| **OCR** | OpenCV + 相关库 | 屏幕内容识别 |
| **监控** | Prometheus + Grafana | 性能监控与告警 |

#### 2.2.2 主要依赖库

- `psutil`：系统信息采集
- `tiktoken`：Token 计数
- `pyyaml`：YAML 配置
- `chromadb`：向量数据库
- `sentence-transformers`：语义向量化
- `opencv-python-headless`：图像处理
- `pyttsx3`：语音合成
- `SpeechRecognition`：语音识别

### 2.3 核心技术组件

#### 2.3.1 分层架构

1. **感知层 (Sensor Layer)**
   - `BodySensor`：传感器管理
   - 实时系统状态采集
   - 事件监控与变化检测

2. **认知层 (Cognitive Layer)**
   - `PromptInjector`：提示词管理
   - 传感器数据拟人化翻译
   - 配置管理与规则引擎

3. **记忆层 (Memory Layer)**
   - `MemoryManager`：记忆管理
   - `BlackBox`：黑匣子日志
   - `VectorStore`：向量记忆
   - `LifeTrace`：三层记忆树

4. **行动层 (Agent Layer)**
   - `DigitalLife`：数字生命主类
   - `BehaviorController`：行为控制
   - `PermissionSystem`：权限系统
   - 工具系统与规划引擎

#### 2.3.2 核心流程

**完整对话流程**：
```
用户输入
    │
    ▼
┌──────────────────┐
│ 感知身体状态    │ ← BodySensor
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 评估是否拒绝    │ ← BehaviorController
└──────────────────┘
    │
    ├─ 拒绝 → 生成拒绝响应 → 用户
    │
    ▼ (正常)
┌──────────────────┐
│ 构建提示词      │ ← PromptInjector
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 获取记忆上下文  │ ← MemoryManager
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 调用 LLM 生成响应│ ← LLMService
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 自我反思        │ ← Reflection
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 保存对话记录    │ ← MemoryManager
└──────────────────┘
    │
    ▼
返回响应给用户
```

## 7. 实现细节

### 7.1 关键技术要点

#### 7.1.1 异步压缩机制

**问题**: 记忆压缩可能耗时较长，阻塞主流程

**解决方案**:
- 使用单独后台线程处理压缩
- 定时检查是否需要压缩
- 支持手动请求压缩
- 同步兜底机制（后台线程未启动时）

**实现**: `AsyncCompressor` 类

```python
class AsyncCompressor:
    def __init__(self, memory_manager, interval=60):
        self._interval = interval
        self._thread = None
        self._stop_event = threading.Event()
    
    def _run(self):
        while not self._stop_event.is_set():
            triggered = self._event.wait(timeout=self._interval)
            if triggered:
                self._do_compress()
```

#### 7.1.2 P5 懒加载优化

**问题**: V2 模块初始化耗时较长，影响启动速度

**解决方案**:
- 延迟初始化到首次访问时
- 记录初始化性能
- 优雅降级（模块不可用时禁用功能）

**实现**:
```python
def _ensure_lifetrace(self):
    """确保 LifeTrace 已初始化"""
    if not self._lifetrace_initialized:
        # 首次访问时初始化
        self._trace_recorder = TraceRecorder(...)
        self._memory_retriever = MemoryRetriever(...)
        self._lifetrace_initialized = True
```

#### 7.1.3 状态快照机制

**问题**: 保存和恢复完整系统状态复杂

**解决方案**:
- 序列化所有关键数据
- 支持增量快照
- gzip 压缩减小体积
- 元数据管理快照版本

**实现**: `StateSnapshotManager` 类

#### 7.1.4 安全执行与超时

**问题**: LLM 调用可能超时或出错

**解决方案**:
- 超时机制（信号 + 线程双保险）
- 异常捕获与兜底响应
- 执行监控防止死循环
- 状态卡死检测

**实现**: `safe_chat` 函数 + `ExecutionMonitor` 类

### 7.2 技术难点与解决方案

| 难点 | 解决方案 |
|-----|---------|
| 记忆无限增长 | 滚动摘要 + 后台压缩 |

| 启动速度慢 | 懒加载 + 并行初始化 |
| 状态恢复复杂 | 快照系统 + 增量优化 |
| 危险操作识别 | 规则匹配 + 二次确认 |

| 长期记忆检索 | 向量数据库 + 语义搜索 |
