# 灵犀界面布局重构设计文档

## 概述

重构灵犀数字生命体 Web 界面的整体布局，解决当前信息堆砌、空间利用低效、导航割裂的问题。保留所有现有内容和功能，仅重新组织空间分配和交互模式。

## 当前问题

1. **侧边栏与主内容割裂**：280px Agent 管理侧边栏与主区域的对话/全景视图没有视觉关联
2. **全景视图内容堆砌**：流水线 + 4 张可展开卡片 + 系统总览 + 交互追踪全部纵向堆叠
3. **对话视图顶部空间浪费**：健康指标栏 + 状态栏占约 100px 高度
4. **缺少统一仪表盘入口**：监控信息分散在对话/全景/侧边栏三处
5. **panorama.css 已有更好的布局但未在 HTML 中使用**

## 设计目标

- 对话和系统监控"两者并重"，同一视图可同时看到
- 管理模块（历史/技能/工具/人格/记忆）低频使用，默认折叠
- 全景视图采用"总览优先，按需深入"模式
- 最大化内容区域可用空间

## 整体布局

```
┌──────────────────────────────────────────────────────────┐
│  Topbar: [☰] 灵犀 · 数字生命体      [设置] [刷新] [清空] │
├──────────────────────────────────────────────────────────┤
│  Tabs: 💬 对话 | 🗺 全景    CPU 42% | 内存 58% | ...     │
├────┬─────────────────────────────────┬────────────────────┤
│ 图 │  主内容区                        │  状态面板 (170px)   │
│ 标 │  (对话视图 / 全景视图)           │  仅对话视图显示     │
│ 栏 │                                 │  - 健康指标         │
│ 48 │                                 │  - 事件流           │
│ px │                                 │                    │
└────┴─────────────────────────────────┴────────────────────┘
```

### 布局方式

CSS Grid，三列结构：
- `grid-template-columns: 48px 1fr 170px`（对话视图）
- `grid-template-columns: 48px 1fr`（全景视图，右侧面板隐藏）

## 组件详细设计

### 1. 顶栏 (Topbar)

保持不变。标题 + 副标题 + 操作按钮（设置/刷新/清空对话）。

### 2. 标签栏 (Tabs)

**变更**：健康指标从独立的 `#chat-metrics-bar` 移动到标签栏右侧。

```
💬 对话 | 🗺 全景          CPU 42% | 内存 58% | 磁盘 72% | 电池 85% | 📡 3/5
```

- 指标显示为紧凑单行，用 `|` 分隔
- 颜色根据严重程度变化（normal=#3fb950, warning=#d29922, critical=#f85149）
- 原 `#chat-metrics-bar` 和 `#chat-status-bar` 删除，释放约 100px 垂直空间
- 健康指标移到标签栏右侧（紧凑单行），模式徽章移到右侧状态面板

### 3. 图标栏 (48px)

替代原 280px 侧边栏。纵向排列 5 个管理模块图标：

```
🕐  历史会话
🔧  技能管理
🛠  工具集成
🎭  人格配置
🧠  记忆管理
```

**交互**：
- 悬停图标 → 右侧显示 tooltip 标签
- 点击图标 → 在图标右侧弹出 240px 浮层面板（含完整功能）
- 点击浮层外部 / ✕ / 再次点击图标 → 关闭浮层
- 同一时间只有一个浮层打开
- 底部 ☰ 按钮 → 展开 280px 完整侧边栏（代替图标栏），再次点击收缩

**浮层面板内容**：将现有侧边栏面板 DOM（`#panel-history` 等）移入浮层容器，复用已有的 JS 加载逻辑（`loadHistory()` 等）。浮层关闭时面板 DOM 保留在浮层容器中，下次打开直接显示。

## 数据流

```
/api/health      → 标签栏指标 + 右侧状态面板 + 全景仪表盘
/api/sensors     → 标签栏传感器计数 + 全景感知层
/api/mode        → 状态徽章 + 右侧面板模式
/api/cognitive   → 全景认知层
/api/panorama    → 全景全部数据（保持现有结构）
/api/chat        → 聊天消息（不变）
/api/history     → 历史浮层
/api/personality → 人格浮层
/api/skills      → 技能浮层
/api/tools       → 工具浮层
/api/memory      → 记忆浮层
```

所有 API 端点保持不变。前端数据获取逻辑调整为从各组件发起请求。

## 响应式策略

| 断点 | 行为 |
|------|------|
| > 900px | 完整三列布局 |
| 768–900px | 右侧状态面板隐藏，可切换显示 |
| 600–768px | 图标栏隐藏，☰ 移到顶栏 |
| < 600px | 全景导航折叠为顶部横排 tab |

## 实现要点

1. **HTML 结构重构**：将内联 `<style>` 提取到独立 CSS 文件，调整 DOM 层次
2. **图标栏浮层**：JS 控制浮层显示/隐藏，点击外部关闭
3. **全景导航切换**：`switchPanoSection()` 函数控制右侧内容区
4. **状态面板**：`updateStatusPanel()` 函数每 10 秒刷新
5. **向后兼容**：所有现有 API 接口不变，数据格式不变
