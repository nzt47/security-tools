## 第一部分：项目背景与核心设计思想

### 1.1 项目立项初心

设计一个高度灵活、支持异构模型调用、具备热更新能力的多 Agent 系统。核心诉求：

1. 主 Agent 保持最基本的核心和开发功能，可建立分身，分身进行装配。
2. 分身包括所有部分自己选配：LLM、记忆管理、工具等，由提供商提供 MCP 调用、本地原生或由主 Agent 临时独立开发的组件。
3. 分身可以热更新。
4. 主 Agent 轻量化、分身高度定制化。

### 1.2 十大核心原则（不可逾越的设计红线）

| # | 原则 | 含义 |
|---|------|------|
| 1 | 透明性 | 对所有系统操作、决策流程及数据流向具有清晰的可见性 |
| 2 | 日志记录 | 完善的日志机制，捕获所有系统活动、错误及用户交互，并附带时间戳与上下文信息 |
| 3 | 可追溯性 | 整个系统生命周期内，所有操作、数据转换及决策均具备完整的可追溯性 |
| 4 | 端到端功能 | 覆盖从输入到输出全流程的端到端能力，尽可能减少对外部依赖 |
| 5 | 本地可用性 | 支持在本地环境中完整运行，无需强制依赖互联网连接或云服务 |
| 6 | 解耦设计 | 模块化架构，明确划分关注点，各组件松耦合且可独立维护 |
| 7 | 杜绝硬编码 | 消除硬编码的数值、凭证及配置参数，改用规范的配置管理机制 |
| 8 | 工作流优先 | 优先通过本地工作流实现功能；仅在严格必要时才将 LLM 作为次要备用方案 |
| 9 | 自解释 UI | 采用自解释 UI，让用户对系统行为一目了然 |
| 10 | 全面测试 | 写完代码必须进行全面测试，保证可用 |

### 1.3 架构哲学（一句话总纲）

"以本地工作流为骨架、以大模型为辅助、以全链路审计为神经、以声明式配置为血液"的工业级智能系统。

把复杂留给后台，把简单交给用户。主 Agent 作为轻量级"指挥官"，通过标准化 MCP 协议和动态编排能力，像搭积木一样按需召唤、装配、更新各种"特种分身"。

### 1.4 完整 Agent 的六大组成部分（每个分身都必须具备）

1. **大脑（模型/LLM）**：推理引擎，处理信息、评估选项、理解任务并做出决策。
2. **编排层（神经系统）**：管理运行循环，包括规划（任务拆解）、记忆管理和推理策略执行。
3. **工具（双手）**：连接外部世界的机制，允许执行文本生成以外的动作（读写文件、搜索网络、执行代码、调用 API）。
4. **记忆模块（海马体）**：维持上下文和持续学习。短期记忆（当前任务上下文）+ 长期记忆（历史数据和用户行为）。
5. **部署（身体）**：托管在安全、可扩展的服务器上，集成监控、日志和可观测性。
6. **前端交互层**：用户与 Agent 系统的交互界面，支持 Web、APP 等多设备适配。

- 敏感凭证的安全注入机制（环境变量 / KMS）

**具体执行任务**：
1. 搭建项目基础目录结构：严格按照第九部分的目录树创建项目骨架，在每个 Python 目录下生成 __init__.py。
2. 创建配置示例模板：
   - configs/models.yaml：本地/云端模型配置示例

6. 实现结构化审计日志系统：在 observability/logger.py 中实现 Append-only 日志，包含时间戳、输入输出哈希、调用栈深度。

**核心交付物**：

**具体执行任务**：

- 基于任务特征的记忆智能路由（Router）

**具体执行任务**：
1. 定义统一的记忆抽象层接口：在 memory/base.py 中定义标准接口（save(), search(), get_profile(), update_graph()），确保与底层提供商解耦。
2. 完成 Holographic 适配器：在 memory/holographic_adapter.py 中实现本地 SQLite 事实存储 + FTS5 搜索 + 信任评分，无需 API Key。
3. 完成 Mem0 适配器：在 memory/mem0_adapter.py 中实现服务端 LLM 事实提取 + 语义搜索 + 自动去重。
4. 预留其余 5 个适配器骨架：在 memory/hindsight_adapter.py、honcho_adapter.py、openviking_adapter.py、retaindb_adapter.py、supermemory_adapter.py 中实现接口骨架与"需人工核实"标注。

6. 实现降级与容错：云端提供商不可用时自动降级到本地 Holographic。
7. 实现工作记忆与持久化记忆分离：工作记忆随分身销毁；持久化记忆通过 memory_update 写回提供商。

   - 双 Agent 校验（Actor-Critic 架构）

3. 实现 Trace 数据订阅接口：在 observability/trace_subscriber.py 中提供供前端实时订阅的 API 或 WebSocket 接口，支持将复杂的黑盒推理过程转化为结构化数据流。
4. 实现多维评估监控：任务完成率、平均重试次数、幻觉率、记忆检索准确率；引入轻量级"评估 Agent"定期抽样打分。
