## P5 具体执行任务

### 1. 开发前端自解释 UI 面板

在 `ui/` 目录下编写前端代码（React + TypeScript 推荐）。要求直观展示任务流转节点与 Agent 思考过程。

### 2. 实现节点状态差异化展示

UI 需清晰区分并高亮展示"工作流节点"与"LLM 兜底节点"的切换状态，让用户对系统行为一目了然。

### 3. 实现实时通信渲染

对接 P4 阶段提供的 Trace 数据接口（WebSocket），实现系统状态的实时渲染与更新。

### 4. 实现执行轨迹可视化

将复杂的黑盒推理过程转化为直观的节点图或时间轴。

### 5. 实现自解释性

UI 直接映射后端执行轨迹，实时展示工作流流转节点、当前调用的工具、模型输出的中间推理过程。

## 📎 设计参考：自解释 UI（第五部分 5.5 节）

> **以下设计规范为 P5 阶段的核心指导原则，执行时必须严格遵循。**

### 5.5 自解释 UI（Self-Explanatory UI）

- 前端界面应直接映射后端的执行轨迹。
- 当 Agent 执行复杂任务时，UI 上能实时展示工作流的流转节点、当前正在调用的工具、以及模型输出的中间推理过程。

**UI 目录结构参考（来自第九部分）：**

```
ui/
├── package.json
├── src/
│   ├── App.tsx                   # 主应用
│   ├── components/
│   │   ├── WorkflowGraph.tsx     # 工作流节点图
│   │   ├── LLMFallbackBadge.tsx  # LLM兜底节点高亮
│   │   └── TraceTimeline.tsx     # 执行轨迹时间轴
│   └── hooks/
│       └── useTraceStream.ts     # Trace实时订阅Hook
└── tests/
```

**UI 组件设计要点：**

1. **WorkflowGraph.tsx**：将任务流转可视化为节点图，节点颜色/形状区分"工作流节点"与"LLM 兜底节点"
2. **LLMFallbackBadge.tsx**：当系统从"工作流优先"切换到"LLM 兜底"时，高亮展示切换状态
3. **TraceTimeline.tsx**：将执行轨迹展示为时间轴，包含每个节点的耗时、Token 消耗、决策路径
4. **useTraceStream.ts**：通过 WebSocket 实时订阅 Trace 数据流，驱动 UI 实时更新
