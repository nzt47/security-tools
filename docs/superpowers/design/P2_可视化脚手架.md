> 完成后输出：可视化编辑器组件 + 代码生成引擎 + 单元/集成测试。

## 1. 背景与目标

### 1.1 现状

`yunshu-ui/` 现有技能管理组件以表单/列表为主：

```
src/components/SkillsMgmt/
├── SkillManagement.tsx    # 主面板（Tab 切换）
├── SkillList.tsx          # 技能列表
├── SkillCreator.tsx       # 技能创建（表单）
├── SkillReviewer.tsx      # 技能审查
├── SkillDetail.tsx        # 技能详情
├── WorkflowMatcher.tsx    # 工作流匹配
└── WorkflowRepo.tsx       # 工作流仓库
```

**缺失**：无可视化拖拽编辑器，用户需手写 YAML/JSON 定义工作流，门槛高、效率低。

### 1.2 目标

- **零代码**：通过拖拽节点 + 连线编排工作流，无需编写任何代码
- **实时预览**：画布右侧实时生成可执行的工作流定义（YAML）
- **组件复用**：已注册的技能自动出现在组件面板中，可直接拖入画布
- **导入/导出**：支持从 YAML 导入已有工作流，编辑后导出
- **响应式**：适配桌面端（移动端降级为只读模式）

### 1.3 不变量（不易）

- 现有 `SkillManagement` 组件结构不破坏（新增 Tab，不改已有 Tab）
- 生成的工作流 YAML 格式与现有 `WorkflowMatcher` 兼容
- Zustand 状态管理范式不变（不引入 Redux/MobX）
- Tailwind CSS 样式体系不变

## 2. 技术选型

### 2.1 选型对比

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| **@xyflow/react (ReactFlow)** | 节点编辑器标杆、社区活跃、自定义节点灵活、TS 原生支持 | 体积较大(~150KB gzip) | **✅ 选用** |
| react-dnd | 轻量、灵活 | 需自行实现画布/连线/缩放，工作量大 | ❌ |
| @antv/x6 | 功能强大、中文文档好 | AntV 生态绑定、TS 支持弱 | ❌ |
| form-based | 零新依赖 | 非真正"可视化拖拽"，不满足需求 | ❌ |

### 3.3 状态管理（Zustand）

```typescript
// stores/useFlowStore.ts
import { create } from 'zustand';
import {
  Node, Edge, OnNodesChange, OnEdgesChange,
  applyNodeChanges, applyEdgeChanges,
} from '@xyflow/react';
import { generateYaml } from '../generator/CodeGenerator';

interface FlowState {
  // ─── 画布数据 ───
  nodes: Node<FlowNodeData>[];
  edges: Edge[];

  // ─── UI 状态 ───
  selectedNodeId: string | null;
  yamlPreview: string;
  dirty: boolean;

  // ─── 操作 ───
  onNodesChange: OnNodesChange;
  onEdgesChange: OnEdgesChange;
  addNode: (node: Node<FlowNodeData>) => void;
  updateNodeData: (id: string, data: Partial<FlowNodeData>) => void;
  removeNode: (id: string) => void;
  selectNode: (id: string | null) => void;

  // ─── 代码生成 ───
  regenerateYaml: () => void;

  // ─── 导入/导出 ───
  importFromYaml: (yaml: string) => void;
  exportYaml: () => string;
}

export const useFlowStore = create<FlowState>((set, get) => ({
  nodes: [],
  edges: [],
  selectedNodeId: null,
  yamlPreview: '',
  dirty: false,

  onNodesChange: (changes) =>
    set((s) => ({ nodes: applyNodeChanges(changes, s.nodes), dirty: true })),

  onEdgesChange: (changes) =>
    set((s) => ({ edges: applyEdgeChanges(changes, s.edges), dirty: true })),

  addNode: (node) =>
    set((s) => ({
      nodes: [...s.nodes, node],
      dirty: true,
      selectedNodeId: node.id,
    })),

  updateNodeData: (id, data) =>
    set((s) => ({
      nodes: s.nodes.map((n) =>
        n.id === id ? { ...n, data: { ...n.data, ...data } } : n
      ),
      dirty: true,
    })),

  removeNode: (id) =>
    set((s) => ({
      nodes: s.nodes.filter((n) => n.id !== id),
      edges: s.edges.filter((e) => e.source !== id && e.target !== id),
      selectedNodeId: s.selectedNodeId === id ? null : s.selectedNodeId,
      dirty: true,
    })),

  selectNode: (id) => set({ selectedNodeId: id }),

  regenerateYaml: () =>
    set((s) => ({ yamlPreview: generateYaml(s.nodes, s.edges) })),

  importFromYaml: (yaml) => {
    const { nodes, edges } = parseYamlToGraph(yaml);
    set({ nodes, edges, dirty: false, yamlPreview: yaml });
  },

  exportYaml: () => get().yamlPreview,
}));
```

```tsx
import { ReactFlow, type OnDrop } from '@xyflow/react';

function FlowCanvas() {
  const { nodes, edges, onNodesChange, onEdgesChange, addNode } = useFlowStore();

  const onDrop: OnDrop = (event) => {
    event.preventDefault();
    const data = event.dataTransfer.getData('application/reactflow');
    const item = JSON.parse(data);

    const position = screenToFlowPosition({
      x: event.clientX,
      y: event.clientY,
    });

    const newNode: Node<FlowNodeData> = {
      id: `${item.type}-${Date.now()}`,
      type: item.type,
      position,
      data: {
        label: item.label,
        nodeType: item.type,
        skillId: item.skillId,
        skillName: item.label,
      },
    };
    addNode(newNode);
  };

  return (
    <ReactFlow
      nodes={nodes}
      edges={edges}
      nodeTypes={nodeTypes}
      onNodesChange={onNodesChange}
      onEdgesChange={onEdgesChange}
      onDrop={onDrop}
      onDragOver={(e) => e.preventDefault()}
      fitView
    />
  );
}
```

## 6. 代码生成引擎

### 6.1 图 → YAML 转换

```typescript
// generator/CodeGenerator.ts
import yaml from 'js-yaml';

export function generateYaml(
  nodes: Node<FlowNodeData>[],
  edges: Edge[]
): string {
  // 1. 拓扑排序（按依赖关系排列节点）
  const sorted = topologicalSort(nodes, edges);

  // 2. 构建 AST
  const ast: WorkflowDefinition = {
    name: 'generated_workflow',
    version: '1.0',
    steps: sorted.map((node) => nodeToStep(node, edges)),
  };

  // 3. 序列化为 YAML
  return yaml.dump(ast, { indent: 2, lineWidth: 80 });
}

function nodeToStep(
  node: Node<FlowNodeData>,
  edges: Edge[]
): WorkflowStep {
  const data = node.data;
  const nextNodes = edges
    .filter((e) => e.source === node.id)
    .map((e) => nodes.find((n) => n.id === e.target)?.data.label)
    .filter(Boolean);

  switch (data.nodeType) {
    case 'skill':
      return {
        name: data.label,
        type: 'skill',
        skill_id: data.skillId,
        params: data.params || {},
        next: nextNodes[0],
        timeout: data.timeout,
        retry: data.retryCount,
      };
    case 'conditional':
      return {
        name: data.label,
        type: 'conditional',
        condition: data.condition,
        true_branch: data.trueBranch,
        false_branch: data.falseBranch,
      };
    case 'loop':
      return {
        name: data.label,
        type: 'loop',
        count: data.loopCount,
        variable: data.loopVariable,
        body: nextNodes[0],
      };
    // ...
  }
}
```

### 6.3 反向解析（YAML → 图）

```typescript
// generator/YamlParser.ts
export function parseYamlToGraph(yaml: string): {
  nodes: Node<FlowNodeData>[];
  edges: Edge[];
} {
  const def = yaml.load(yaml) as WorkflowDefinition;
  const nodes: Node<FlowNodeData>[] = [];
  const edges: Edge[] = [];

  for (const step of def.steps) {
    const node = stepToNode(step);
    nodes.push(node);

    // 根据 next/true_branch/false_branch 构建 edges
    if (step.next) {
      const target = nodes.find((n) => n.data.label === step.next);
      if (target) edges.push({ source: node.id, target: target.id });
    }
    // ...
  }
  return { nodes, edges };
}
```

## 7. 属性面板（PropertiesPanel）

```tsx
// components/PropertiesPanel.tsx
function PropertiesPanel() {
  const { nodes, selectedNodeId, updateNodeData, removeNode } = useFlowStore();
  const node = nodes.find((n) => n.id === selectedNodeId);

  if (!node) {
    return <div className="properties-empty">选择节点以编辑属性</div>;
  }

  const { data } = node;

  return (
    <div className="properties-panel">
      <h3>属性</h3>

      <div className="prop-group">
        <label>名称</label>
        <input
          value={data.label}
          onChange={(e) => updateNodeData(node.id, { label: e.target.value })}
        />
      </div>

      {data.nodeType === 'skill' && (
        <>
          <div className="prop-group">
            <label>技能 ID</label>
            <input value={data.skillId || ''} readOnly />
          </div>
          <div className="prop-group">
            <label>超时(秒)</label>
            <input
              type="number"
              value={data.timeout || 30}
              onChange={(e) => updateNodeData(node.id, { timeout: +e.target.value })}
            />
          </div>
          <div className="prop-group">
            <label>重试次数</label>
            <input
              type="number"
              value={data.retryCount || 0}
              onChange={(e) => updateNodeData(node.id, { retryCount: +e.target.value })}
            />
          </div>
        </>
      )}

      {data.nodeType === 'conditional' && (
        <div className="prop-group">
          <label>条件表达式</label>
          <textarea
            value={data.condition || ''}
            onChange={(e) => updateNodeData(node.id, { condition: e.target.value })}
            placeholder="page_count > 10"
          />
        </div>
      )}

      <button className="btn-danger" onClick={() => removeNode(node.id)}>
        删除节点
      </button>
    </div>
  );
}
```

### 12.3 工作流冲突解决策略

工作流 YAML 的冲突解决与 skill.md 不同：skill.md 按 front matter 字段合并，工作流按 `step.name` 逐步骤合并。

| 场景 | 合并策略 | 示例 |
|------|----------|------|
| 步骤仅在一方存在 | 直接保留 | A 新增"格式化"步骤，B 无变更 → 保留 |
| 同名步骤双方都修改 | 字段级合并 | A 改 timeout，B 改 params → 合并两者 |
| 同名步骤一方删除 | 保留删除方的决定 | A 删除"校验"步骤，B 修改该步骤 → 删除 |
| 步骤顺序冲突 | 按 A 的顺序为准 | A 调整顺序，B 新增步骤 → A 顺序 + B 新增 |

### 12.4 WorkflowConflictResolver 类设计

```python
# agent/skills_mgmt/workflow_conflict_resolver.py
from typing import Any, Optional
from .conflict_resolver import ConflictResolver

class WorkflowConflictResolver(ConflictResolver):
    """工作流 YAML 冲突解决器，按 step.name 逐步骤合并。"""

    def _merge_field(self, key: str, ours: Any, theirs: Any) -> Optional[Any]:
        if key == "steps" and isinstance(ours, list) and isinstance(theirs, list):
            return self._merge_steps(ours, theirs)
        return super()._merge_field(key, ours, theirs)

    def _merge_steps(self, ours: list, theirs: list) -> Optional[list]:
        ours_map = {s["name"]: s for s in ours if "name" in s}
        theirs_map = {s["name"]: s for s in theirs if "name" in s}
        all_names = list(ours_map.keys()) + [
            n for n in theirs_map if n not in ours_map
        ]
        merged = []
        for name in all_names:
            if name in ours_map and name in theirs_map:
                field_merged = self._merge_step_fields(ours_map[name], theirs_map[name])
                if field_merged is None:
                    return None
                merged.append(field_merged)
            elif name in ours_map:
                merged.append(ours_map[name])
            else:
                merged.append(theirs_map[name])
        return merged

    def _merge_step_fields(self, ours: dict, theirs: dict) -> Optional[dict]:
        merged = {}
        for key in set(list(ours.keys()) + list(theirs.keys())):
            if key in ours and key in theirs:
                if ours[key] == theirs[key]:
                    merged[key] = ours[key]
                else:
                    field = self._merge_field(key, ours[key], theirs[key])
                    if field is None:
                        return None
                    merged[key] = field
            elif key in ours:
                merged[key] = ours[key]
            else:
                merged[key] = theirs[key]
        return merged
```

```typescript
export function validateNode(node: Node<FlowNodeData>): ValidationResult {
  const rules = NODE_RULES[node.data.nodeType];
  const errors: string[] = [];
  for (const rule of rules) {
    const value = node.data[rule.field as keyof FlowNodeData];
    if (rule.required && (value === undefined || value === null)) {
      errors.push(rule.message);
      continue;
    }
    if (value === undefined) continue;
    if (rule.type === 'number') {
      const num = Number(value);
      if (isNaN(num)) { errors.push(rule.message); continue; }
      if (rule.min !== undefined && num < rule.min) errors.push(rule.message);
      if (rule.max !== undefined && num > rule.max) errors.push(rule.message);
    }
  }
  return { valid: errors.length === 0, errors };
}
```

### 13.4 TopologyValidator

```typescript
// validator/TopologyValidator.ts
export interface TopologyResult {
  valid: boolean;
  errors: string[];
  warnings: string[];
}

export function validateTopology(
  nodes: Node<FlowNodeData>[],
  edges: Edge[]
): TopologyResult {
  const errors: string[] = [];
  const warnings: string[] = [];

  // 1. 检查孤立节点（无入边也无出边）
  const connectedIds = new Set<string>();
  edges.forEach(e => { connectedIds.add(e.source); connectedIds.add(e.target); });
  const orphans = nodes.filter(n => !connectedIds.has(n.id));
  if (orphans.length > 0) {
    warnings.push(`孤立节点: ${orphans.map(n => n.data.label).join(', ')}`);
  }

  // 2. 检查环路（DFS）
  const adjList = new Map<string, string[]>();
  edges.forEach(e => {
    if (!adjList.has(e.source)) adjList.set(e.source, []);
    adjList.get(e.source)!.push(e.target);
  });
  const visited = new Set<string>();
  const recursionStack = new Set<string>();
  for (const node of nodes) {
    if (hasCycle(node.id, adjList, visited, recursionStack)) {
      errors.push('检测到环路，工作流不能包含循环依赖');
      break;
    }
  }

  // 3. 检查条件节点分支完整性
  const nodeMap = new Map(nodes.map(n => [n.id, n]));
  for (const node of nodes) {
    if (node.data.nodeType === 'conditional') {
      if (!node.data.trueBranch || !node.data.falseBranch) {
        errors.push(`条件节点"${node.data.label}"缺少 True/False 分支`);
      }
    }
  }

  return { valid: errors.length === 0, errors, warnings };
}

function hasCycle(
  nodeId: string,
  adjList: Map<string, string[]>,
  visited: Set<string>,
  stack: Set<string>
): boolean {
  if (stack.has(nodeId)) return true;
  if (visited.has(nodeId)) return false;
  visited.add(nodeId);
  stack.add(nodeId);
  const neighbors = adjList.get(nodeId) || [];
  for (const next of neighbors) {
    if (hasCycle(next, adjList, visited, stack)) return true;
  }
  stack.delete(nodeId);
  return false;
}
```
