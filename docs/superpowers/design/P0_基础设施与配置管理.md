### 2. 创建配置示例模板

- **configs/models.yaml**：本地/云端模型配置示例
- **configs/memory.yaml**：记忆提供商路由配置示例
- **configs/security.yaml**：权限与护栏配置示例

### 3. 实现声明式配置加载模块

编写配置读取器（`core/config_loader.py`），支持 YAML/JSON 格式解析，确保系统无硬编码。

### 4. 实现敏感凭证的安全注入机制

通过环境变量（`os.environ`）或 KMS 安全加载凭证，严禁在代码中写死。

### 5. 实现全局 Trace_ID 生成器

在 `observability/tracer.py` 中实现 Trace_ID 生成与传播机制。

### 6. 实现结构化审计日志系统

在 `observability/logger.py` 中实现 Append-only 日志，包含时间戳、输入输出哈希、调用栈深度。

### 7. 创建 main.py 系统入口

能加载配置、初始化日志、生成第一条 Trace_ID 并打印启动日志。

## 📎 设计参考：声明式配置管理（第八部分 8.4 节）

> **以下设计规范为 P0 阶段的核心指导原则，执行时必须严格遵循。**

所有的数值、凭证、模型路径、工具权限等，一律通过 YAML/JSON 配置文件驱动。

敏感凭证（如 API Key、数据库密码）必须通过密钥管理系统（KMS）或环境变量动态加载，严禁硬编码。

工具调用必须在配置文件中显式声明允许的路径和权限（如 `allowed_paths`），超出范围直接报错，而非静默失败。
