## 4. 关键发现

### 4.1 性能维度

1. **500 节点 generateYaml 远低于验收线**：实测 6.4-27ms（波动由 JIT 预热导致），远低于 100ms 验收线。原预测 200ms 过于保守，无需 Web Worker 降级。

2. **所有 UI 操作 0 次卡顿**：500 节点下所有 store→rerender 端到端耗时 <4ms，远低于 16ms 流畅线（60fps）。

3. **浅引用快照极速**：undo/redo 仅切换数组引用（0.27-1.25ms），不涉及深拷贝，这是浅引用策略的核心优势。

### 4.2 内存维度

1. **浅引用策略有效**：50 个快照仅增 0.23-0.27MB，而非理论最坏值 10.5MB。未变更节点对象在快照间共享。

2. **无累积泄漏**：250 次 undo/redo 循环后增量 0.02-0.7MB（无 GC 时波动），有 GC 时增量 0.02MB。

3. **redoStack 清空有效**：新操作触发 `redoStack.length = 0` 后，快照引用被释放，GC 可回收。

### 4.3 边界安全维度

1. **无栈溢出风险**：`undo`/`redo` 在空栈时 `return`，1000 次连续调用安全。

2. **MAX_HISTORY=50 硬上限有效**：超过 50 步历史时自动 `shift()`，栈深度永不超限。

3. **数据一致性保证**：50 次 undo + 50 次 redo 后，节点数/edges/数据均与原始状态一致。
