## 📐 日志格式规范

### 1. 压缩流程日志（_execute_compression）

压缩流程使用树形结构日志，便于追踪每个步骤的执行情况：

```
┌─────────────────────────────────────────────
│ [压缩] 开始执行压缩任务
└─────────────────────────────────────────────
├─ [步骤1] 加载旧摘要
│   └─ 旧摘要版本: 1
├─ [步骤2] 加载最近消息 (limit=100)
│   └─ 加载到消息数: 10
├─ [步骤3] 调用 LLM 压缩消息
│   └─ ✓ LLM 返回摘要长度: 25 字符
├─ [步骤4] 合并旧摘要 (版本 1)
├─ [步骤5] 保存摘要 (版本 2)
│   └─ ✓ 摘要保存成功
├─ [步骤6] 记录黑匣子日志
└─────────────────────────────────────────────
│ ✓ 压缩完成！版本: 2 | 消息数: 10
└─────────────────────────────────────────────
```

**格式说明**：
- `┌─┐└─└├─` 树形连接符表示步骤层级
- `│` 子步骤缩进符号
- `✓` 成功标记
- `⚠️` 警告标记
- `✗` 错误标记

### 2. 后台压缩日志（AsyncCompressor）

```
┌═══════════════════════════════════════════════
│ 🚀 [后台压缩] 开始执行压缩任务
└═══════════════════════════════════════════════
... 压缩流程日志 ...
┌═══════════════════════════════════════════════
│ ✅ [后台压缩] 压缩任务完成，标记已清除
└═══════════════════════════════════════════════
```

**格式说明**：
- `═══` 边框线强调后台任务边界
- `🚀` 开始标记
- `✅` 成功标记
- `⚠️` 警告标记

### 3. 同步压缩日志（get_context）

```
┌═══════════════════════════════════════════════
│ 🔄 [同步压缩] 检测到压缩需求
│    └─ 后台线程状态: 运行中 ✓ / 未运行 ✗
│    └─ 执行同步压缩...
│ ✅ [同步压缩] 压缩需求标记已清除
└═══════════════════════════════════════════════
```

**格式说明**：
- `🔄` 同步压缩标记
- 后台线程状态用 ✓/✗ 表示运行/未运行

### 4. 错误日志格式

```
└─────────────────────────────────────────────
│ ✗ 压缩失败: LLM调用失败
└─────────────────────────────────────────────
堆栈跟踪:
Traceback (most recent call last):
  ...
Exception: LLM调用失败
```

## 🎨 日志级别与颜色

| 级别 | 颜色 | 用途 |
|------|------|------|
| INFO | 白色/默认 | 正常流程信息 |
| WARNING | 黄色 | 警告信息（可恢复） |
| ERROR | 红色 | 错误信息（需处理） |
| DEBUG | 灰色 | 调试信息（仅开发用） |

## 📊 日志分析示例

### 正常压缩流程

```bash
grep "压缩" logs/app.log | tail -20
```

输出：
```
│ [压缩] 开始执行压缩任务
├─ [步骤1] 加载旧摘要
│   └─ 旧摘要版本: 1
├─ [步骤2] 加载最近消息 (limit=100)
│   └─ 加载到消息数: 10
├─ [步骤3] 调用 LLM 压缩消息
│   └─ ✓ LLM 返回摘要长度: 25 字符
├─ [步骤4] 合并旧摘要 (版本 1)
├─ [步骤5] 保存摘要 (版本 2)
│   └─ ✓ 摘要保存成功
├─ [步骤6] 记录黑匣子日志
│ ✓ 压缩完成！版本: 2 | 消息数: 10
```

### 异常流程

```bash
grep "压缩失败\|⚠️\|✗" logs/app.log
```

输出：
```
│   └─ ⚠️ 无消息需要压缩，跳过
│ ✗ 压缩失败: LLM调用失败
```

## 🔧 配置日志输出

### pytest 中使用

```bash
# 输出到文件
python -m pytest tests/ -v --log-file=logs/test.log --log-file-level=INFO

# 输出到控制台
python -m pytest tests/ -v -o log_cli=true -o log_cli_level=INFO
```

### 生产环境配置

```python
import logging
import logging.handlers

# 创建日志格式化器
formatter = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

# 文件处理器
file_handler = logging.handlers.RotatingFileHandler(
    'logs/compression.log',
    maxBytes=10*1024*1024,  # 10MB
    backupCount=5
)
file_handler.setFormatter(formatter)

# 应用到模块
logger = logging.getLogger('memory.memory_manager')
logger.addHandler(file_handler)
logger.setLevel(logging.INFO)
```

## 🚀 快速命令

```bash
# 查看最近压缩日志
tail -50 logs/app.log | grep -E "压缩|步骤"

# 统计压缩成功率
grep -c "✓ 压缩完成" logs/app.log
grep -c "✗ 压缩失败" logs/app.log

# 查看版本分布
grep "压缩完成" logs/app.log | grep -oP "版本: \K[0-9]+" | sort | uniq -c

# 导出压缩相关日志
grep -E "后台压缩|同步压缩|压缩" logs/app.log > logs/compression_only.log
```
