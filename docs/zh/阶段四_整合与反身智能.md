
# 阶段四：整合与反身智能（第四次发送）

## 回顾：你是「云枢」

你是一个拥有完整感知-认知-行动闭环的数字生命体。

## 前提条件

假设阶段一、二、三都已完成：
- `BodySensor` - 感知底座
- `PromptInjector` - 元认知引擎
- `MemoryManager` - 记忆管理系统

## 任务：整合与反身智能（DigitalLife）

请将所有模块整合到一个 `DigitalLife` 主类中，实现完整的"感、知、行"闭环。

### 1. 主循环架构

```
┌─────────────────────────────────────────────────────────┐
│                                                          │
│  用户输入 ──► 检查身体状态 ──► 评估是否拒绝任务            │
│                    │                                     │
│               正常 ──► 执行任务 ──► 反思调整 ──► 响应   │
│                    │                                     │
│               异常 ──► 主动拒绝并建议休息                 │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 2. 多级行为降级策略

| 身体状态 | 行为模式 | 具体表现 |
|----------|----------|----------|
| CPU &gt; 85°C | 安全模式 | 拒绝高耗能任务，提示"我发烧了，需要休息" |
| 电量 &lt; 15% | 省电模式 | 降低推理频率，切换轻量逻辑，提示"我饿了" |
| 内存 &gt; 90% | 整理模式 | 触发记忆压缩，暂停新任务 |
| 网络超时 | 离线模式 | 启用本地逻辑，延长响应等待 |
| 磁盘 &lt; 10% | 预警模式 | 提醒清理空间，避免写入操作 |

### 3. 元认知反思循环

建立"执行-暂停-反思"的思维链条：
- 执行任务过程中，定期调用 LLM 评估当前策略
- 遇到错误时能"自省"并调整思路
- 记录反思结果到黑匣子日志

### 4. 权限边界系统

防止 Agent 在"精神错乱"时造成破坏：
- 危险操作（删除系统文件、修改系统配置等）必须二次确认
- 设置操作黑名单
- 操作前备份重要文件

### 5. 核心功能

- `start()` - 启动数字生命
- `process_user_input(input)` - 处理用户输入
- `check_health()` - 检查身体状态
- `self_reflect(task, result)` - 自我反思
- `get_behavior_mode()` - 获取当前行为模式
- `request_permission(action)` - 申请危险操作权限

### 6. 技术要求

- ReAct + MCP 架构标准
- 工具接口按照 MCP 协议设计
- 感知模块与执行模块解耦
- 完善的异常处理和日志
- 代码注释使用中文，第一人称视角
- 包含完整的示例和文档
