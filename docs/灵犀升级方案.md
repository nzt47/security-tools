## 🧠 模块一：LifeTrace - 海马体记忆系统

### 设计理念

参考 **OpenHuman 的 Memory Tree** 架构，实现三层记忆树：
- **原子记忆** (Source Tree)：原始数据，按来源分类
- **主题记忆** (Topic Tree)：按主题聚类的记忆
- **全局记忆** (Global Tree)：长期摘要和核心人格

### 数据结构设计

```
lifetrace/
├── data/
│   ├── sources/          # 来源树
│   │   ├── chat/         # 对话记录
│   │   ├── sensor/       # 传感器数据
│   │   ├── window/       # 窗口活动
│   │   └── file/         # 文件变更
│   ├── topics/           # 主题树
│   │   ├── work/         # 工作相关
│   │   ├── personal/     # 个人偏好
│   │   └── learn/        # 学习内容
│   └── global/           # 全局树
│       ├── persona.json  # 核心人格
│       └── summary.md    # 长期摘要
└── index/
    ├── vector.db         # 向量索引
    └── tag.db            # 标签索引
```

### 核心类设计

```python
# lifetrace/memory_tree.py
class MemoryTree:
    """三层记忆树架构"""

class SourceTree(MemoryTree):
    """来源树 - 原始数据存储"""

class TopicTree(MemoryTree):
    """主题树 - 聚类记忆"""

class GlobalTree(MemoryTree):
    """全局树 - 人格和摘要"""

# lifetrace/trace_recorder.py
class TraceRecorder:
    """多维度数据采集器"""

    def record_chat(self, role, content):
        """记录对话"""

    def record_sensor(self, sensor_data):
        """记录传感器"""

    def record_window(self, window_event):
        """记录窗口活动"""

    def record_file(self, file_event):
        """记录文件变更"""
```

### 数据采集维度

| 维度 | 数据源 | 采集频率 |
|------|--------|----------|
| 对话交互 | 用户对话、系统响应 | 实时 |
| 身体感知 | CPU、内存、电池、温度 | 每分钟 |
| 环境感知 | 窗口切换、活动应用 | 事件驱动 |
| 行为痕迹 | 文件变更、代码编辑 | 事件驱动 |
| 学习记录 | 新知识、技能获取 | 人工标注 |

## 🌐 模块三：OpenHuman/AIRI 风格交互界面

### 设计理念

融合 **AIRI 的具身交互** 和 **OpenHuman 的桌面常驻**：
- Live2D/VRM 虚拟形象
- 桌面 Mascot 吉祥物
- 主动感知和介入
- 声音/表情/动作反馈

### UI 组件设计

#### 1. 主界面 - 三层布局

```
┌───────────────────────────────────────────────────────┐
│  🤖 云枢 Mascot        │  对话区域               │
│  (Live2D 动画)          │                        │
│                        │  ┌──────────────────┐  │
│  [系统状态]            │  │  消息列表        │  │
│  • CPU: 45%            │  └──────────────────┘  │
│  • 内存: 60%           │                        │
│  • 电池: 85%           │  [输入框] [发送]       │
└───────────────────────────────────────────────────────┘
```

#### 2. Mascot 状态系统

| 状态 | 动画/表情 | 触发条件 |
|------|-----------|----------|
| 😊 正常 | 呼吸动画、眨眼 | 身体状态良好 |
| 🥵 发热 | 冒汗动画、皱眉 | CPU &gt; 85°C |
| 😴 省电 | 闭眼、缓慢动作 | 电池 &lt; 20% |
| 🤔 思考 | 托腮、眼球转动 | 正在生成响应 |
| 😊 开心 | 微笑、点头 | 用户表扬/正面反馈 |

#### 3. 主动交互模式

参考 OpenHuman 的潜意识系统，实现后台主动服务：

```python
# 主动服务场景
- 检测到用户持续工作 2 小时 → "需要休息一下吗？"
- 检测到 CPU 过热 → "我感觉有点热，要不要帮你关掉些进程？"
- 检测到新文件创建 → "这是个什么文件？需要我帮你整理吗？"
- 定期记忆回顾 → "记得你昨天在学 Python，有进展吗？"
```
