## 三、瓶颈定位

### 主要瓶颈分析

| 模块 | 初始化耗时 | 占比 | 瓶颈等级 | 说明 |
|------|----------|------|---------|------|
| **ChangeDetector** | 18.173ms | 6.3% | 🟡 中 | 加载历史变更记录（3212条） |
| **EventMonitor** | 268.029ms | **93.4%** | 🔴 **高** | detect_startup_changes() 导致 |
| FileWatcher | 未初始化 | - | 🟢 低 | 无 watch_dirs 配置 |
| **传感器自发现** | ~237ms | **82%** | 🟡 中 | 注册 18 个传感器 |

### 🔴 关键瓶颈：EventMonitor 初始化（占 93.4%）

**问题描述**:
- EventMonitor 在初始化时调用 `detect_startup_changes()` 方法
- 该方法需要检测所有硬件设备的变化
- 耗时高达 268.029ms，占总初始化时间的 93.4%

**优化建议**:
1. **延迟初始化**：将 `detect_startup_changes()` 延迟到后台执行
2. **异步处理**：使用异步方式检测启动变化
3. **增量检测**：只检测上次关机后的变化，而不是全量扫描

## 六、进一步优化建议（基于瓶颈分析）

### 优先级 1：EventMonitor 异步初始化
**问题**: EventMonitor 占用了 93.4% 的初始化时间
**建议**:
```python
# 当前实现（阻塞）
def _init_event_monitor(self):
    self.event_monitor.start()
    # 同步调用 detect_startup_changes()
    startup_changes = self.event_monitor.detect_startup_changes()

# 优化后（异步）
async def _init_event_monitor_async(self):
    self.event_monitor.start()
    # 异步调用，不阻塞初始化
    asyncio.create_task(self._detect_startup_changes_async())
```

### 优先级 2：传感器自发现优化
**问题**: 自发现 18 个传感器耗时 ~237ms
**建议**:
- 分类加载：核心传感器立即加载，非核心传感器懒加载
- 缓存机制：缓存已发现的传感器列表

### 优先级 3：ChangeDetector 历史记录优化
**问题**: 加载 3212 条历史变更记录耗时 18.173ms
**建议**:
- 分页加载：只加载最近的 N 条记录
- 异步加载：在后台加载完整历史
