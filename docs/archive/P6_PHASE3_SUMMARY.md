## 一、Phase 3 新增内容

### 1. 核心模块恢复方法

已为所有核心模块实现了完整的恢复逻辑：

#### (1) BodySensor 恢复 (`_restore_body_sensor`)
- 恢复初始化状态 (`_initialized`)
- 恢复观察目录配置 (`watch_dirs`)
- 恢复传感器配置 (`config`)
- 异常处理与详细日志

#### (2) BehaviorController 恢复 (`_restore_behavior`)
- 恢复当前行为模式 (`_current_mode`)
- 恢复模式历史记录 (`_mode_history`)
- 枚举模式兼容性检查
- 异常处理与详细日志

#### (3) PermissionSystem 恢复 (`_restore_permission`)
- 恢复危险模式统计
- 恢复敏感文件扩展名列表
- 安全配置记录
- 异常处理与详细日志

#### (4) ToolsRegistry 恢复 (`_restore_tools_registry`)
- 恢复工具注册状态
- 恢复工具列表（最多50个）
- 工具统计记录
- 异常处理与详细日志

### 2. 按优先级模块恢复 (`_restore_modules_by_priority`)

实现了智能的模块恢复调度器：
- 按优先级顺序恢复模块（BodySensor → Behavior → Permission → Tools）
- 数据完整性校验（SHA-256 checksum）
- 单个模块失败不影响其他模块恢复
- 详细的恢复进度日志
- 恢复成功率统计

### 3. 更新的 `load_snapshot` 方法

增强的快照恢复方法，支持两种模式：

**向后兼容模式（Phase 1-2）**：
```python
# 仅加载快照数据
snapshot = manager.load_snapshot()  # 返回 StateSnapshot 对象
```

**完整实例恢复（Phase 3）**：
```python
# 完整恢复 DigitalLife 实例
Yunshu = manager.load_snapshot(
    digital_life_class=DigitalLife,  # 提供类对象
    snapshot_id="snap_xxx"           # 可选，默认最新快照
)
```

恢复流程包含6个关键步骤：
1. **定位快照文件** - 查找并加载快照
2. **版本兼容性检查** - 验证快照格式兼容
3. **创建实例** - 使用快照配置初始化
4. **优先级模块恢复** - 按顺序恢复核心模块
5. **恢复验证** - 检查核心模块完整性
6. **更新管理器状态** - 缓存快照引用

## 三、main_p6.py 入口文件

### 1. 功能特性

- ✅ **快照优先启动** - 默认尝试从快照恢复
- ✅ **频率控制** - 支持强制保存和正常保存
- ✅ **清理命令** - `--cleanup` 清理所有快照
- ✅ **状态查看** - `--status` 查看当前状态
- ✅ **单次对话** - `--chat` 一次性对话模式
- ✅ **交互式模式** - 完整的交互式体验
- ✅ **退出时自动保存** - 自动保存状态快照

### 2. 使用示例

```bash
# 正常启动（优先从快照恢复）
python main_p6.py

# 禁用快照，使用正常初始化
python main_p6.py --no-snapshot

# 强制保存快照（忽略频率限制）
python main_p6.py --force-save

# 查看状态后退出
python main_p6.py --status

# 单次对话模式
python main_p6.py --chat "你好"

# 清理所有快照
python main_p6.py --cleanup
```
