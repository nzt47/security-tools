## 系统架构总览

### 核心模块

#### 1. 感知层 (Sensor Layer)
```
├─ BodySensor        身体感知（CPU、GPU、内存、电池等）
├─ WindowSensor      窗口活动感知
├─ OcrSensor         屏幕内容OCR识别
└─ EventMonitor      系统事件监听
```

#### 2. 记忆层 (Memory Layer)
```
├─ LifeTrace         三层记忆系统（来源树/主题树/全局树）
│  ├─ SourceTree     原始数据记录
│  ├─ TopicTree      主题聚类组织
│  └─ GlobalTree     全局摘要和人格
├─ TraceRecorder     数据记录器
├─ EnhancedTraceRecorder 增强记录器
└─ MemoryRetriever   智能记忆检索
```

#### 3. 人格层 (Persona Layer)
```
├─ PersonaModel      五层人格模型（immortal-skill架构）
├─ PersonaInjector   人格提示词注入器
└─ PersonalityPreferenceExtractor 人格蒸馏器
   ├─ 表达风格分析
   ├─ 话题兴趣识别
   ├─ 交互时间模式
   └─ 持续学习机制
```

#### 4. 认知层 (Cognitive Layer)
```
├─ PromptInjector    旧版提示词注入器（兼容）
├─ BehaviorController 行为控制器
└─ PermissionSystem 权限系统
```

#### 5. 行动层 (Action Layer)
```
├─ Tool System       工具系统
└─ LLM Integration   大语言模型集成
```

### 数据流转
```
用户输入
    ↓
感知层（传感器采集）
    ↓
记录层（LifeTrace存储）
    ↓
认知层（人格蒸馏 + 提示词生成）
    ↓
判断层（权限 + 行为模式评估）
    ↓
行动层（LLM推理 + 工具调用）
    ↓
反思层（元认知 + 自我优化）
    ↓
学习层（人格蒸馏持续更新）
    ↓
响应输出
```

## 核心功能亮点

### 1. 人格蒸馏系统
- **持续学习**: 每N次交互自动学习用户偏好
- **多维分析**: 风格、话题、时间、工具四维分析
- **动态适配**: 根据学习结果自动调整回应风格
- **持久化存储**: 偏好数据自动保存和加载

### 2. LifeTrace记忆系统
- **三层树结构**: 来源树、主题树、全局树
- **智能检索**: 语义搜索相关记忆
- **无限记忆**: 自动摘要 + 滑动窗口
- **增强记录**: 支持窗口活动、OCR内容记录

### 3. Persona人格系统
- **五层架构**: 规则、身份、风格、决策、交互
- **提示词注入**: 动态生成个性化系统提示
- **决策辅助**: 人格参与任务决策

### 4. 感知系统
- **全面感知**: 身体、窗口、屏幕、事件
- **事件驱动**: 支持热插拔、文件变更等事件
- **低开销**: 智能轮询机制，不影响系统性能

## 使用指南

### 快速开始

#### 1. 基本使用
```python

# 初始化并启动
Yunshu = DigitalLifeV2()

# 对话
response = Yunshu.chat("你好！")
print(response)

# 获取状态
status = Yunshu.get_status()

# 停止
Yunshu.stop()
```
