## 📋 一、问题概述

### 1.1 问题描述
用户反馈云枢的历史记忆不能保存，具体表现为**对话不保存到记忆**。

### 1.2 根因分析
| 层级 | 状态 | 说明 |
|------|------|------|
| `data/messages.jsonl`（文件持久化） | ✅ 正常 | `MemoryManager.add_message` 正确写入文件 |
| `_CHAT_HISTORY`（内存列表） | ❌ 问题源 | 服务器重启后清空，前端 `/api/history` 只读取此列表 |
| 启动加载逻辑 | ❌ 缺失 | 无代码从持久化文件加载历史到内存 |

**结论**: 记忆保存链路完整，但重启后**内存与文件之间的桥接缺失**，导致前端无法显示历史。

## 🛠️ 二、修复方案

### 2.1 修复内容
在 `app_server.py` 中新增 `_load_chat_history_from_file()` 函数：

```python
def _load_chat_history_from_file():
    """从持久化文件加载历史对话到内存（带容错和详细日志）"""
```

### 2.2 调用时机
在 `DigitalLife` 实例化并启动后、服务器接收请求前调用：


### 2.3 容错机制设计

| 容错场景 | 处理策略 | 日志级别 |
|----------|----------|----------|
| 文件不存在 | 跳过加载，空列表 | WARNING |
| 文件大小无法获取 | 跳过加载 | ERROR |
| 单行 JSON 解析失败 | 跳过该行，继续处理 | WARNING |
| 行缺少必要字段 | 跳过该行 | WARNING |
| 文件编码错误 (utf-8) | 尝试 utf-8-sig 备用编码 | ERROR → INFO |
| 备用编码也失败 | 清空列表，安全退出 | ERROR |
| 文件中无有效消息 | 空列表 | WARNING |
| 消息不匹配（user/assistant） | 跳过当前消息继续尝试 | DEBUG |
| 最后一条消息未配对 | 跳过 | DEBUG |
