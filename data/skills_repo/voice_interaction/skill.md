# 语音交互

通过语音输入与输出与用户进行交互，支持语音转文字识别、文字转语音合成与语音指令路由，适用于免手操作与无障碍场景。

## 适用场景

- 用户不便打字时，通过语音输入指令与对话内容
- 视觉障碍用户通过语音播报获取回应
- 长文本回应场景下，自动转语音播放减轻阅读负担

## 触发条件

- 用户显式启用语音模式（"开启语音 / 切换语音输入"）
- 检测到音频输入设备连接且用户开始说话
- 回应超过一定长度（默认 200 字）且启用自动播报时

## 使用方式

1. 语音输入：采集音频 → ASR 转文字 → 进入对话处理流程
2. 语音指令路由：识别"停止 / 暂停 / 继续 / 切换主题"等控制指令
3. 语音输出：回应文本 → TTS 合成 → 播放音频流
4. 中断处理：用户说话时若再次发声，触发打断并停止当前播报
5. 错误恢复：ASR / TTS 失败时回退至文本模式并提示

## 输出特征

- `asr_text`: 识别出的文本
- `tts_audio_url`: 合成音频的访问地址（或二进制流）
- `command`: 识别到的控制指令（如无则为 null）
- `interrupted`: 是否被用户打断（bool）

## 不变量

- 语音数据不得在本地持久化明文存储，处理完即丢弃
- ASR 识别结果必须回显给用户确认，不可静默执行
- 打断响应延迟须 ≤ 500ms，避免信息继续输出造成混淆
- TTS 合成失败时必须回退文本模式，不得静默丢弃回应
