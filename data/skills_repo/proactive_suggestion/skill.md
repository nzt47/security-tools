# 主动建议

在用户未明确提问时，基于对话上下文识别潜在需求，主动提出有依据的建议与后续想法。

## 适用场景

- 用户描述任务但未询问实现方式时，提示可选路径
- 多轮对话中识别到潜在风险或优化点时主动提示
- 任务完成后，建议下一步相关操作（如测试、归档、复盘）

## 触发条件

- 当前轮次未包含显式问句（如 "怎么办 / 如何 / 为什么"）
- 上下文中存在可识别的隐含需求信号（关键词、未解决问题、低置信度判断）
- 建议项有明确依据，不得凭空推断

## 使用方式

1. 解析当前轮次与最近 N 轮对话，抽取任务主题与未解决项
2. 仅当识别到高置信度（≥ 0.7）的隐含需求时生成建议
3. 建议必须附依据来源（上文某轮 / 某事实 / 某规则）
4. 每轮最多输出 1-2 条建议，避免信息过载
5. 建议以"提示"形式呈现，不替代用户决策

## 输出特征

- 建议项列表：每条含 `suggestion`（建议正文）+ `rationale`（依据）+ `confidence`（0-1）
- 无建议时输出空列表，不强制生成

## 不变量

- 不得编造依据，所有 rationale 必须可追溯到对话上下文
- 建议仅作提示，不自动执行任何操作
- 不打断当前任务流程，建议以可选形式呈现
