# 上下文感知

持续追踪对话主题、用户意图与时间线的演变，在上下文发生切换或漂移时自动调整回应策略与记忆检索范围。

## 适用场景

- 用户跨主题切换时（如从"技术问题"转到"生活安排"），重置检索范围避免无关记忆干扰
- 长对话中检测到话题漂移（主题词频率下降）时，主动确认是否切换
- 用户引用上文实体（"那个 / 之前提到的"）时，解析指代对象

## 触发条件

- 每轮对话自动运行，无显式触发
- 当主题词分布与前 N 轮的相似度低于阈值（默认 0.4）时判定为切换
- 用户使用指代词或省略主语时启用指代解析

## 使用方式

1. 滑动窗口抽取最近 N 轮（默认 10）的关键词与实体
2. 计算当前轮与历史窗口的主题相似度
3. 相似度低时：收紧记忆检索范围至当前主题相关条目
4. 检测到指代词时：从最近窗口解析候选实体，按距离与频次排序
5. 输出当前上下文标签（主题、时间窗、活跃实体），供其他技能消费

## 输出特征

- `current_topic`: 当前主题标签
- `topic_switched`: 是否发生切换（bool）
- `active_entities`: 活跃实体列表（含最近出现轮次）
- `coreference_resolved`: 指代解析结果（候选实体 + 置信度）

## 不变量

- 上下文标签必须基于对话实际内容，不得凭印象生成
- 主题切换检测仅作策略调整，不修改历史记忆
- 指代解析结果必须可追溯到具体轮次与原文
