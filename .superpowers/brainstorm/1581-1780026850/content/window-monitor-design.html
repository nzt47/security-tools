方案 A：WindowSensor + BlackBox + 记忆面板扩展

1. 架构总览

系统架构
🪟

获取前台窗口句柄
提取标题 + 进程名
↓

event_type:
"window_event"

save_log()
query_logs()
↓
GET /api/memory/windows/events

时长/频率/排行
↓
🧠 记忆管理面板 → 窗口活动分页

间隔 1s | 保留 500 条

2. 数据模型

BlackBox 日志中的窗口事件格式
{
  "id": "bb_0123",
  "timestamp": "2026-05-29T14:32:05",
  "event_type": "window_event",
  "data": {
    "action": "switch",           // switch | focus | blur | idle
    "from_window": "VS Code",     // 上一个窗口标题（首次为 null）
    "from_process": "Code.exe",
    "to_window": "Google Chrome",
    "to_process": "chrome.exe",
    "duration_sec": 245           // 在上一窗口停留的秒数
  }
}
统计聚合（stats API 返回）

{
  "total_duration": 3600,
  "apps": [
    { "process": "chrome.exe",
      "title": "Google Chrome",
      "duration_sec": 1620,
      "switch_count": 12,
      "percentage": 45 }
  ]
}
配置参数（data/window_config.json）

{
  "enabled": true,
  "poll_interval_sec": 1,
  "max_events": 500,
  "idle_timeout_sec": 300,
  "ignore_processes": ["Idle"]
}
