诊断发现的 5 个核心问题

1. 侧边栏与主内容割裂

左侧 280px 的 Agent 管理侧边栏包含 5 个管理模块（历史/技能/工具/人格/记忆），但它们与主区域的「对话」和「全景」视图没有视觉或功能上的关联，左右各自独立滚动，用户认知负担重。

2. 全景视图内容堆砌

全景视图将所有内容（流水线 + 4 张可展开卡片 + 系统总览 + 交互追踪）全部纵向堆叠在一个滚动区域中，没有侧导航或分栏，查找信息需要大量滚动。

3. 对话视图顶部空间浪费

健康指标栏 + 状态栏在对话视图顶部占约 100px 高度，压缩了下方消息列表的可视空间。指标虽是实时数据但排列松散。

4. 缺少统一的仪表盘入口

用户需要在「对话」看健康指标、在「全景」看架构详情、在「侧边栏」管理配置——三处信息本应关联，却被物理隔离在不同区域。

5. panorama.css 已有更好的布局设计但未使用

panorama.css 中定义了 .pano-layout / .pano-nav / .pano-content 的左导航+右内容布局，以及 .pano-healthbar 横向健康条，但 index.html 并未采用，仍使用最简单的 .pano-scroll 垂直堆叠。
