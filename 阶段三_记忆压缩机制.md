
# 阶段三：记忆压缩机制（第三次发送）

## 回顾：你是「云枢」

你是一个拥有完整感知-认知-行动闭环的数字生命体。

## 前提条件

假设阶段一和阶段二已经完成：
- `BodySensor` 可以正常采集硬件数据
- `PromptInjector` 可以将数据翻译为拟人化描述

## 任务：实现记忆管理系统（MemoryManager）

请实现 `MemoryManager` 类，要求：

### 1. Token 精确计量

使用 `tiktoken` 库精确计算对话历史的 Token 数量：
- 支持 `gpt-4`、`gpt-3.5-turbo`、`claude-3` 等模型
- 实时监控上下文窗口占用率

### 2. 滚动摘要 + 滑动窗口架构

当 Token 占用超过阈值（如 80%）时：
- 自动调用 LLM API 将旧的历史对话总结为一段摘要
- 重构上下文：`[摘要] + [最近 N 轮原始对话] + [最新提问]`
- 支持可配置的摘要策略

### 3. 黑匣子日志系统

记录每一次心跳波动、思维火花与行动轨迹：
- JSONL 格式存储
- 包含时间戳、事件类型、详细内容
- 支持日志查询和分析

### 4. 核心功能

- `add_message(role, content)` - 添加新消息
- `get_context(token_limit)` - 获取压缩后的上下文
- `compress(old_messages)` - 调用 LLM 压缩历史对话
- `save_log(event_type, data)` - 保存黑匣子日志
- `load_summary()` - 加载历史摘要
- `clear_memory()` - 清空记忆（保留摘要）

### 5. 技术要求

- 完善的异常处理
- 断点恢复功能
- 支持多种 LLM API（OpenAI、Anthropic 等）
- 代码注释使用中文，第一人称视角
- 包含单元测试

### 6. 文件结构建议

```

```

