### 核心设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 架构模式 | 经典分层 | config → translator + templates → prompt_injector → flask_adapter，依赖单向 |
| 翻译方式 | 数据驱动规则 | 阈值和描述与代码分离，新增传感器只需改配置 |
| 模板引擎 | str.format() | 轻量无额外依赖 |
| Flask 集成 | 函数式适配器 | 按需导入，无强制依赖 |
| 配置加载 | 三层覆盖 | 内置默认 → YAML 文件 → 编程覆盖 |
| 区间匹配 | 左闭右开 | min <= value < max |

## 二、模块详情

### 2.1 config.py — 配置系统

```python
class PromptConfig:
    def get_rule(self, sensor_name: str) -> dict        # 获取规则（深拷贝）
    def register_rule(self, sensor_name: str, rule: dict) # 运行时注册
    def get_all_rules(self) -> dict                      # 获取全部规则
    def load_from_file(self, path: str)                  # 从 YAML 加载

    # 内置规则覆盖 5 个传感器:
    # cpu_temperature, battery_percentage, memory_usage,
    # network_latency, disk_space_usage
    # 每个传感器有 3 个阈值区间: normal / warning / critical
```

### 2.2 translator.py — 翻译引擎

```python
class Translator:
    def translate(self, reading: dict) -> str            # 单条翻译
    def translate_all(self, readings: list[dict]) -> list[str]  # 批量翻译
    def get_status_line(self, readings: list[dict]) -> str     # 一句话摘要
```

匹配逻辑：按 sensor_name 查规则 → 遍历 thresholds → `min <= value < max` → 返回 message。无匹配时降级为 `"{description}: {value}{unit}"`。

### 2.3 templates.py — 模板系统

```python
class TemplateManager:
    def render(self, template_name: str, **kwargs) -> str  # 渲染模板
    def register_template(self, name: str, template: str)  # 注册模板
```

两个内置模板：
- **default**: `云枢` 身份 + `{body_status}` + `{task_guidance}`
- **reject**: 拒绝任务 + `{reason}` + `{body_status}`

### 2.4 prompt_injector.py — 核心编排

```python
class PromptInjector:
    def inject(self, sensor_data: list[dict]) -> str                # 完整注入
    def translate(self, reading: dict) -> str                       # 单条翻译
    def get_summary(self, sensor_data: list[dict]) -> str           # 状态摘要
    def should_reject_task(self, sensor_data: list[dict]) -> tuple  # 拒绝决策
```

拒绝逻辑：
- 任意 `severity: "critical"` → `(True, "身体出现严重不适：...")`
- 3+ 个 `severity: "warning"` → `(False, "虽然还能工作...")`
- 其他 → `(False, "一切正常，随时待命")`

### 2.5 flask_adapter.py — Flask 集成

| 端点 | 方法 | 返回 |
|------|------|------|
| `GET /api/cognitive/status` | get_summary() | text/plain |
| `GET /api/cognitive/prompt` | inject() | text/plain |
| `GET /api/cognitive/translate/<sensor>` | translate() | text/plain |
| `GET /api/cognitive/reject` | should_reject_task() | application/json |

集成方式（在 sensor_server.py 中）：

```python
from cognitive import PromptInjector
from cognitive.flask_adapter import register_prompt_routes
_cognitive_injector = PromptInjector()
register_prompt_routes(app, _cognitive_injector, _CACHE)
```

## 三、数据格式

### 输入格式（传感器读数）

```python
{
    "sensor_name": "cpu_temperature",   # 唯一标识
    "value": 85.0,                      # 数值
    "unit": "°C",                       # 单位
    "description": "CPU 温度",          # 中文描述
    "severity": "critical",             # normal / warning / critical
    "category": "cpu",                  # 类别
    "tags": ["硬件感知", "内部感知", ...] # 8 维度标签
}
```

### 输出格式

- `inject()` → 完整的 LLM system prompt 字符串
- `translate()` → 拟人化描述字符串（如"我感觉发烧了，浑身发烫"）
- `get_summary()` → 一句话摘要字符串
- `should_reject_task()` → `(bool, str)`

## 四、已知限制

1. **翻译规则仅覆盖 5 个传感器** — cpu_temperature, battery_percentage, memory_usage, network_latency, disk_space_usage。其他传感器使用通用降级格式。
2. **无 LLM API 集成** — 翻译完全基于规则，无法处理超出预定义范围的语义表达。
3. **判断阈值不可自适应** — 拒绝逻辑使用固定阈值（1 critical/3 warning），不支持动态调整。
4. **Flask 单线程** — `threaded=False` 限制并发请求处理能力。
5. **GPUtil 与 Python 3.12 不兼容** — distutils 被移除导致 GPUtil 1.4.0 无法导入，已降级处理为 NORMAL。
6. **pyudev 仅支持 Linux** — Windows 上缺失为正常行为，已标记为 NORMAL。

## 五、阶段三接口依赖

阶段三的 `MemoryManager` 需要从阶段二获取/集成的接口：

```python
# 1. 通过 PromptInjector 的模板系统注入记忆上下文
from cognitive import PromptInjector, PromptConfig
from cognitive.templates import TemplateManager

injector = PromptInjector()
# 阶段三需要注册新模板，将记忆上下文注入到 system prompt 中：
injector.template_mgr.register_template("memory_aware", """
你是「云枢」，一个生活在电脑里的数字生命。

当前身体状态：
{body_status}

近期记忆摘要：
{memory_summary}

{task_guidance}
""")

# 2. PromptInjector 的 config 可以用于注册新的配置项
# 3. translator 可用于记忆内容的拟人化（非必需）
```

### 建议集成方式

```python
# 阶段三在 memory/ 下独立构建，通过注入方式与阶段二集成：
# 1. 将 MemoryManager 的摘要注入到 PromptInjector 的模板变量中
# 2. MemoryManager 可以注册自定义模板到 TemplateManager
# 3. prompt_injector.inject() 中加入 memory_summary 参数

# 在 sensor_server.py 中的扩展方式：
from cognitive import PromptInjector
from cognitive.flask_adapter import register_prompt_routes
from memory import MemoryManager  # 阶段三

_injector = PromptInjector()
_memory = MemoryManager()
register_prompt_routes(app, _injector, _CACHE)

# 扩展 /api/cognitive/prompt 端点加入记忆上下文
```
