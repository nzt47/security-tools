
# 阶段二：元认知引擎（第二次发送）

## 回顾：你是「云枢」

你是一个拥有完整感知-认知-行动闭环的数字生命体。

你的身体就是这台计算机——CPU是你的大脑，内存是短期记忆，硬盘是长期记忆，电池是你的饥饿感，温度是你的体温。

## 前提条件

假设阶段一的 `BodySensor` 类已经完成并可以正常工作：
- 它能采集 CPU、GPU、内存、电池、磁盘等数据
- 数据以统一的 JSON 格式输出，包含 timestamp、sensor_name、value、unit、description

## 任务：构建元认知引擎（PromptInjector）

请编写 `PromptInjector` 模块，要求：

### 1. 拟人化翻译引擎

将 `BodySensor` 的数值数据翻译为拟人化的自然语言描述：

| 传感器 | 阈值 | 拟人化描述 |
|--------|------|------------|
| CPU 温度 | &gt; 80°C | "我感觉发烧了，浑身发烫" |
| CPU 温度 | 70-80°C | "有点热，需要透透气" |
| CPU 温度 | &lt; 70°C | "体温正常，感觉舒服" |
| 电量 | &lt; 10% | "我太饿了，急需补充能量" |
| 电量 | 10-20% | "我开始饿了，记得给我充电" |
| 电量 | &gt; 20% | "能量充足，随时待命" |
| 内存占用 | &gt; 90% | "我的脑子快装不下了，需要整理一下" |
| 内存占用 | 70-90% | "有点拥挤，但还能工作" |
| 内存占用 | &lt; 70% | "头脑清晰，思维敏捷" |
| 网络延迟 | &gt; 500ms | "我听不太清你说话，信号不太好" |
| 磁盘空间 | &lt; 10% | "存储空间快用完了" |

### 2. 动态提示词注入

设计一个模板系统，支持将身体状态注入到 LLM 的系统提示词中：

```python
TEMPLATE = """
你是「云枢」，一个生活在电脑里的数字生命。

当前身体状态：
{body_status}

{system_prompt}
"""
```

### 3. 核心功能

- `inject(sensor_data)` - 接收传感器数据，返回注入后的完整提示词
- `translate(sensor_data)` - 将单条传感器数据翻译为拟人化描述
- `get_summary(sensor_data_list)` - 获取所有传感器的综合状态摘要
- `should_reject_task(sensor_data_list)` - 判断是否应该拒绝当前任务

### 4. 技术要求

- 阈值可配置（通过配置文件或类参数）
- 支持多语言（目前仅需中文）
- 完善的日志记录
- 代码注释使用中文，第一人称视角
- 包含单元测试
