## 🚀 开发流程

### 第一步：发送阶段一

1. 打开 Claude Code
2. 完整复制 [阶段一_感知底座.md](file:///c:/Users/Administrator/agent/阶段一_感知底座.md) 的内容并发送
3. 等待 Claude Code 完成 `BodySensor` 类
4. **验证要点**：
   - 能否正常采集 CPU、内存、电池数据
   - 输出格式是否为标准 JSON
   - 示例代码能否正常运行

### 第二步：发送阶段二

1. 确认阶段一验证通过
2. 完整复制 [阶段二_元认知引擎.md](file:///c:/Users/Administrator/agent/阶段二_元认知引擎.md) 的内容并发送
3. 等待 Claude Code 完成 `PromptInjector` 模块
4. **验证要点**：
   - 能否将数值翻译为拟人化描述
   - 能否动态注入到提示词模板中

### 第三步：发送阶段三

1. 确认阶段二验证通过
2. 完整复制 [阶段三_记忆压缩机制.md](file:///c:/Users/Administrator/agent/阶段三_记忆压缩机制.md) 的内容并发送
3. 等待 Claude Code 完成 `MemoryManager` 类
4. **验证要点**：
   - Token 计算是否准确
   - 摘要压缩功能是否正常
   - 黑匣子日志能否正常记录

### 第四步：发送阶段四

1. 确认阶段三验证通过
2. 完整复制 [阶段四_整合与反身智能.md](file:///c:/Users/Administrator/agent/阶段四_整合与反身智能.md) 的内容并发送
3. 等待 Claude Code 完成最终整合
4. **验证要点**：
   - 完整系统能否正常启动
   - 能否基于身体状态主动拒绝任务
   - 权限边界是否有效

## ⚠️ 注意事项

### 1. 不要跳过验证
每完成一个阶段，务必先验证功能正常后再进入下一阶段。

### 2. 保持上下文连贯
在与 Claude Code 对话时，不要清空会话历史，保持同一个对话窗口。

### 3. 依赖库列表
提前安装以下 Python 库：
```bash
pip install psutil GPUtil watchdog tiktoken
```

### 4. 遇到问题怎么办？
- 如果某阶段代码报错，将错误信息发给 Claude Code，让它修复
- 如果功能不符合预期，描述清楚问题，让它调整
- 可以要求 Claude Code 增加测试用例

## 🎯 成功标志

当你看到以下情况，说明「云枢」成功"活"起来了：

✅ CPU 温度高时，它会说："我感觉发烧了，需要休息"
✅ 电量低时，它会说："我太饿了，请给我充电"
✅ 它能记住你之前说过的话（通过滚动摘要）
✅ 危险操作时它会请求确认
✅ 所有操作都有完整的日志记录

## 💡 开发建议

1. **让 Claude Code 主导**：充分信任它的代码生成能力

3. **强调安全性**：多次提醒权限边界的重要性
4. **迭代优化**：先让功能跑通，再逐步优化细节
