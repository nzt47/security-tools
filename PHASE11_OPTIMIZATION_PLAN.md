## 三、第十一阶段重点任务

### 3.1 核心任务：集成测试体系建设

**目标**: 建立 orchestrator 与 memory、tool_calling 模块的集成测试

**测试场景**:

```
┌─────────────────────────────────────────────────────────────┐
│ 集成测试场景规划                                              │
├─────────────────────────────────────────────────────────────┤
│ 1. Orchestrator + Memory                                     │
│    - test_orchestrator_stores_conversation_in_memory        │
│    - test_orchestrator_searches_memory_on_query             │
│    - test_memory_router_fallback_on_adapter_failure          │
│                                                              │
│ 2. Orchestrator + Tool Calling                                │
│    - test_orchestrator_calls_tool_and_handles_result         │
│    - test_tool_failure_propagates_correctly                  │
│    - test_tool_timeout_handling                             │
│                                                              │
│ 3. Memory + Tool Calling                                     │
│    - test_memory_result_triggers_tool_execution              │
│    - test_tool_result_stored_in_memory                       │
└─────────────────────────────────────────────────────────────┘
```

**技术要求**:
- 使用 `pytest-asyncio` 测试异步代码
- Mock 外部依赖（LLM API、数据库）
- 覆盖成功路径和错误路径
- 每个测试用例独立，无隐式依赖

### 3.2 重点任务：性能基准测试体系

**目标**: 建立性能基线，防止重构引入性能退化

**基准指标**:

| 模块 | 指标 | 当前基线 | 目标 | 测量方法 |
|------|------|----------|------|----------|
| LLM 调用 | 平均响应时间 | 待测量 | < 2s | 10次平均 |
| 记忆存储 | save() 延迟 | 待测量 | < 100ms | 100次平均 |
| 记忆检索 | search() 延迟 | 待测量 | < 200ms | 100次平均 |
| 工具调用 | 同步调用延迟 | 待测量 | < 500ms | 50次平均 |
| 编排器 | chat() 端到端 | 待测量 | < 3s | 20次平均 |

**实现方案**:
```python
# performance_benchmark.py
class PerformanceBenchmark:
    """性能基准测试框架"""
    
    def measure_llm_call(self, iterations=10) -> Dict:
        """测量 LLM 调用性能"""
        
    def measure_memory_ops(self, iterations=100) -> Dict:
        """测量记忆操作性能"""
        
    def measure_orchestrator_e2e(self, iterations=20) -> Dict:
        """测量编排器端到端性能"""
        
    def generate_report(self) -> str:
        """生成性能报告"""
```

### 3.3 重点任务：敏感数据过滤统一

**目标**: 统一 `REDACTED_VALUE` 期望值，修复测试失败

**当前问题**:
- `SensitiveDataFilter` 使用 `********` (8个星号)
- 测试期望 `***` (3个星号)
- 导致 4 个测试用例失败

**修复方案**:
```python
# agent/utils/sensitive_data_filter.py
# 统一脱敏值
REDACTED_VALUE = "***"  # 改为 3 个星号，与测试期望一致

# 更新 mask() 方法中的所有替换规则
result = re.sub(..., "***", result)  # 统一使用 "***"
```

**验证**:
```bash
pytest tests/unit/test_config_secure.py::TestSensitiveDataFilter -v
```
