## 它的内部结构长这样

```json
{
    "skills": [],          // 🧠 内建技能
    "claude_skills": [],   // 📂 Claude Skills
    "mcps": [],            // ⚙️ MCP 服务
    "channels": [],        // 🌉 通信通道
    "plugins": []          // 🔌 插件
}
```

**每个扩展条目**包含 `ext_id`、`name`、`description`、`status`（enabled/disabled）、`config` 等字段。

---

## 它是怎么工作的？

1. **系统启动时** → 从 `extensions.json` 读取所有扩展数据
2. **安装/卸载/启用/禁用扩展时** → 实时更新这个 JSON 文件
3. **`ext_list` 查询时** → 我就是从这个文件里读取你的扩展清单的

所以之前我回答你的每一次「列出扩展」——数据源头就在这里 💾
