## 概述

为了方便后续排查问题，我们在规划引擎的核心模块中添加了详细的日志输出。以下是各模块的日志输出指南。

## 日志级别

我们使用标准的 Python logging 模块，默认日志级别为 `INFO`。

```python
import logging
logging.basicConfig(level=logging.INFO)
```

## 日志输出格式

### 标准日志格式
```
LEVEL:logger_name:message
```

例如：
```
INFO:planning.core:🔍 [规划引擎] 开始创建执行计划
WARNING:planning.react:⚠️ 检测到执行循环
ERROR:planning.core:❌ 创建计划异常: Invalid task
```

### 特殊符号说明

- 🔍 分析/查询
- 📋 任务/步骤
- ✅ 成功
- ⚠️ 警告
- ❌ 错误
- 🔄 循环/重复
- 💭 思考
- ⚡ 行动
- 🧠 反思
- 🚀 开始执行
- 📊 统计/数据
- 💬 对话
- 🔧 规则/工具
- 📈 进度

## 如何调试

### 1. 查看完整执行流程

运行测试并查看详细日志：

```bash
python test_planning.py 2>&1 | grep "INFO:planning"
```

### 2. 追踪特定模块

只查看任务分解器的日志：

```bash
python test_planning.py 2>&1 | grep "planning.decomposer"
```

### 3. 查看错误详情

```bash
python test_planning.py 2>&1 | grep -A 5 "ERROR:"
```

### 4. 启用DEBUG级别

如果需要更详细的信息，可以启用DEBUG级别：

```python
import logging

# 设置所有规划引擎模块为DEBUG级别
logging.getLogger('planning.core').setLevel(logging.DEBUG)
logging.getLogger('planning.decomposer').setLevel(logging.DEBUG)
logging.getLogger('planning.react').setLevel(logging.DEBUG)
```

## 常见问题排查

### 问题1: 任务分解失败

查看日志：
```
📊 步骤1: 选择分解策略...
   ⚠️ LLM服务不可用，使用规则降级分解
🔧 [规则分解] 开始基于关键词的任务分解
❌ 任务分解失败: ...
```

解决方案：
- 检查LLM服务是否可用
- 检查规则分解是否能处理该任务
- 查看具体错误信息

### 问题2: 工具调用失败

查看日志：
```
   ⚡ 步骤2: 行动阶段...
   [行动] 工具名: check_cpu
   [行动] 开始调用工具...
   [行动] ❌ 工具执行失败: ...
```

解决方案：
- 检查工具是否已注册
- 检查工具参数是否正确
- 查看异常堆栈跟踪

### 问题3: 状态转换失败

查看日志：
```
❌ 状态转换错误: 无法从 executing 转换到 completed
   当前状态: executing
```

解决方案：
- 检查状态机转换规则
- 查看具体错误信息

### 问题4: 反思执行失败

查看日志：
```
   🧠 步骤3: 反思阶段...
   ⚠️ 反思执行失败: ...
```

解决方案：
- 检查反思引擎配置
- 查看具体错误信息
